## メモリのアドレスとは

コンピュータのメモリは、巨大な棚のようなものです。
それぞれの棚には**番号（アドレス）**がついていて、変数はどこかの棚に格納されています。

```
アドレス:  1000  1001  1002  1003 ...
          ┌────┬────┬────┬────┐
          │ 42 │    │    │    │ ...
          └────┴────┴────┴────┘
               ↑
           int a = 42; はここに入っている
```

**ポインタ** とは、この「アドレスを格納する変数」のことです。
「変数の値」ではなく「変数がどこにあるか」を保持します。

---

## アドレスを取得する（& 演算子）

変数の前に `&` をつけると、その変数のアドレスを取得できます。

```cpp
#include <iostream>
using namespace std;

int main() {
    int a = 42;
    cout << a << endl;    // → 42（値）
    cout << &a << endl;   // → 0x... （アドレス、実行環境によって変わる）
    return 0;
}
```

> 💡 参照（前のページの `int& b = a`）の `&` と、アドレス取得の `&a` は書き方が似ていますが別物です。文脈で使い分けます。

---

## ポインタ変数の宣言

ポインタ変数は型の後に `*` をつけて宣言します。

```cpp
int* p;    // int型のアドレスを格納するポインタ
```

アドレスを代入するには `&` を使います。

```cpp
int a = 42;
int* p = &a;   // p に a のアドレスを格納
```

---

## ポインタが指す値を取得する（* 演算子）

ポインタの前に `*` をつけると、そのアドレスにある値を取得（または変更）できます。
これを **デリファレンス（参照外し）** といいます。

```cpp
#include <iostream>
using namespace std;

int main() {
    int a = 42;
    int* p = &a;   // p は a のアドレスを指す

    cout << p << endl;    // → アドレス（0x...）
    cout << *p << endl;   // → 42（p が指す先の値）

    *p = 99;   // p が指す先の値を変更
    cout << a << endl;    // → 99（a も変わる）

    return 0;
}
```

---

## ポインタと参照の違い

参照とポインタは似ていますが、いくつか違いがあります。

| | 参照（`&`） | ポインタ（`*`） |
|--|------------|----------------|
| 宣言後の変更 | 別の変数を指せない | 別のアドレスを指せる |
| 初期化 | 必須 | 省略可（危険） |
| null | 代入不可 | `nullptr` を代入できる |
| 書き方 | `a` と同じように使える | `*p` と書く必要がある |

**初心者のうちは参照を優先して使い、ポインタは必要になったときに使う**のがおすすめです。

---

## nullptr：何も指さないポインタ

ポインタが何も指していない状態を表すのが `nullptr` です。

```cpp
int* p = nullptr;   // 何も指していない

if (p == nullptr) {
    cout << "ポインタは空です" << endl;
}
```

初期化されていないポインタを使うと予期しないバグになります。
使う前に必ず初期化する習慣をつけましょう。

---

## ポインタを使う場面

現代のC++では、ポインタを直接使う場面は昔より少なくなっています。
ただし、次のような場面で登場します。

- **配列の操作**（配列名はポインタとして扱える）
- **動的メモリ確保**（`new` / `delete`）
- **他のライブラリのAPI**でポインタが要求されるとき

```cpp
// 配列とポインタの関係
int arr[3] = {10, 20, 30};
int* p = arr;   // 配列の先頭アドレスを指す

cout << *p << endl;       // → 10
cout << *(p + 1) << endl; // → 20
cout << *(p + 2) << endl; // → 30
```

---

## まとめ

- 変数はメモリ上の特定のアドレスに格納されている
- `&変数` でアドレスを取得できる
- ポインタ（`型* 変数`）はアドレスを格納する変数
- `*ポインタ` でポインタが指す先の値にアクセスできる
- 何も指さないポインタは `nullptr` で初期化する
- 迷ったら参照を使い、ポインタは必要なときだけ使う
