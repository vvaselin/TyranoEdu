## バグとデバッグ

プログラムが意図した通りに動かないことを **バグ** といいます。
バグを見つけて直す作業を **デバッグ** といいます。

プログラムを書く時間の多くは、実はデバッグに使われます。
効率よくデバッグできると、開発がぐっと楽になります。

---

## エラーの種類

### コンパイルエラー

コンパイル時に検出されるエラーです。
プログラムが実行される前に止まるので、比較的わかりやすいです。

```
error: expected ';' before 'return'
```

→ `;` の書き忘れなど、文法的な間違いが原因です。

### 実行時エラー（ランタイムエラー）

実行中に発生するエラーです。

```
Segmentation fault (core dumped)
```

→ 配列の範囲外アクセス、`nullptr` の参照外しなどが代表的な原因です。

### 論理エラー

エラーは出ないが、**結果が間違っている**状態です。
一番見つけにくいバグです。

```cpp
// 平均を求めたいのに…
int avg = a + b / 2;   // 演算子の優先順位を間違えている
int avg = (a + b) / 2; // こちらが正しい
```

---

## デバッグの基本：`cout` で確認する

最もシンプルなデバッグ方法は、怪しい箇所に `cout` を入れて変数の値を確認することです。

```cpp
int result = calculate(x, y);
cout << "DEBUG: x=" << x << " y=" << y << " result=" << result << endl;
```

「ここまで来た」「この時点での値はこれ」を確認しながら、
期待値と違う場所を絞り込んでいきます。

> 💡 デバッグ用の出力は、問題が解決したら必ず消しましょう。

---

## `cerr` を使う

デバッグ出力には `cout` の代わりに `cerr`（標準エラー出力）を使うと、
通常の出力と分けて管理できます。

```cpp
cerr << "DEBUG: x=" << x << endl;
```

`cerr` はバッファリングされないため、プログラムがクラッシュした直前の出力も確実に表示されます。

---

## よくあるバグと対処法

### 1. 配列・vectorの範囲外アクセス

```cpp
vector<int> v = {1, 2, 3};
cout << v[3] << endl;   // バグ！インデックスは 0〜2 まで
```

**対処:** ループの条件や `size()` を確認する。

---

### 2. 初期化忘れ

```cpp
int sum;
for (int i = 1; i <= 10; i++) {
    sum += i;   // sum の初期値が不定！
}
```

**対処:** 変数は必ず宣言時に初期化する。

```cpp
int sum = 0;   // 初期値を明示
```

---

### 3. `=` と `==` の混同

```cpp
if (x = 5) {   // x に 5 を代入してしまっている
```

**対処:** 比較には `==` を使う。「定数を左に書く」スタイルで防ぐ方法もある。

```cpp
if (5 == x) {   // 誤って = にすると左辺が定数なのでコンパイルエラーになる
```

---

### 4. 整数除算の切り捨て

```cpp
int a = 7, b = 2;
double result = a / b;   // → 3.0（3.5 にならない！）
```

**対処:** 小数が必要なら `double` でキャストする。

```cpp
double result = (double)a / b;   // → 3.5
```

---

### 5. オフバイワンエラー（1つずれ）

ループの回数・配列の範囲が1つずれているバグです。

```cpp
for (int i = 0; i <= v.size(); i++) {   // <= なので1回多い！
```

**対処:** ループの境界条件を `<` か `<=` か慎重に確認する。

---

### 6. 無限ループ

```cpp
while (true) {
    // break がない、または条件が変わらない
}
```

**対処:** ループが必ず終わる条件になっているか確認する。

---

## アサーション（assert）

「この時点でこの条件が必ず成り立つはず」という確認に使います。
条件が成り立たない場合、プログラムが即座に停止してエラーを報告します。

```cpp
#include <cassert>

assert(v.size() > 0);   // v が空でないことを確認
assert(index >= 0 && index < v.size());   // インデックスが有効か確認
```

デバッグ中に「ここは絶対こうなるはず」という前提を明示するのに役立ちます。

---

## デバッグの進め方

1. **再現する** — どういう入力でバグが起きるか確認する
2. **絞り込む** — `cout` でどこまでは正しいか確認する
3. **仮説を立てる** — 「この変数がおかしいのでは？」と予想する
4. **確認する** — 仮説を検証する
5. **修正して再確認** — 修正後に他のケースでも動くか確認する

バグは「必ず原因がある」ので、焦らず順番に絞り込んでいきましょう。

---

## まとめ

- エラーには**コンパイルエラー・実行時エラー・論理エラー**の3種類がある
- `cout` / `cerr` で変数の値を出力して確認するのが基本
- よくあるバグ：範囲外アクセス・初期化忘れ・`=`と`==`の混同・整数除算・オフバイワン
- `assert()` で前提条件を明示できる
- デバッグは「再現→絞り込む→仮説→確認→修正」の流れで
