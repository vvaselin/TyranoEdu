# 第11章-2: 標準入力の高速化

## なぜ高速化が必要？

競技プログラミングや大量のデータを扱うプログラムでは、
入出力の処理がボトルネックになることがあります。

C++のデフォルト設定では、`cin` / `cout` はC言語の `scanf` / `printf` と同期を取るため、
その分のオーバーヘッドが発生しています。

次の2行を `main` の先頭に書くと、入出力が大幅に高速化されます。

---

## 高速化の基本セット

```cpp
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    // 以降、通常通りに cin / cout を使う
}
```

### `ios::sync_with_stdio(false)`

C++ の `cin` / `cout` と、C言語の `scanf` / `printf` の同期を切ります。
これだけで `cin` が数倍速くなります。

> ⚠️ この設定をした後は `scanf` / `printf` と `cin` / `cout` を混在させてはいけません。

### `cin.tie(nullptr)`

デフォルトでは、`cin` が呼ばれるたびに `cout` がフラッシュ（バッファを吐き出す）されます。
`cin.tie(nullptr)` でこの連動を切り、余計なフラッシュをなくします。

---

## `endl` の代わりに `"\n"` を使う

`endl` は改行 + フラッシュを行います。
大量に出力する場面では、改行だけの `"\n"` の方が高速です。

```cpp
// 遅い
cout << "Hello" << endl;

// 速い
cout << "Hello" << "\n";
// または
cout << "Hello\n";
```

---

## まとめてテンプレート

競プロでよく使われる冒頭のテンプレートです。

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    // ここからプログラムを書く

    return 0;
}
```

---

## 実際にどのくらい速くなる？

100万件の整数を読み込む場合の目安：

| 方法 | 速度 |
|------|------|
| `cin`（同期あり） | 遅い |
| `cin`（高速化あり） | 速い（`scanf` と同等程度） |
| `scanf` | 速い |

高速化の設定を入れるだけで、**タイムアウトが解消されることも多い**です。

---

## 注意点まとめ

- `sync_with_stdio(false)` 後は `scanf` / `printf` と混在不可
- `cin.tie(nullptr)` 後は `cin` の前に `cout` が自動フラッシュされない
  → 入力前に出力したいプロンプトは `cout << flush;` か `cerr` を使う
- `"\n"` で十分な場面では `endl` を使わない
