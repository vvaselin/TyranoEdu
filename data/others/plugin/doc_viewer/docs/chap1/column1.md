## 同じ名前がかぶったら？

突然ですが、こんな場面を想像してください。

クラスに「田中くん」が2人いたとします。
先生が「田中くん！」と呼ぶと、どちらの田中くんかわかりません。

そこで先生は言い方を変えます。
「**1組の**田中くん」「**2組の**田中くん」

これで区別できますね。

---

## プログラムでも同じ問題が起きる

プログラムの世界でも、同じ名前の機能が複数存在することがあります。

たとえば `cout` という名前は、C++の標準機能として用意されているものです。
でも、誰かが自分で `cout` という名前の別の機能を作ることもできます。

このとき「どちらの `cout` を使うの？」とコンピュータが混乱しないよう、
**グループ名をつけて区別する**仕組みが用意されています。

このグループのことを **名前空間（namespace）** といいます。

---

## `std` はC++標準機能のグループ名

C++の標準機能はすべて **`std`**（スタンダード）という名前空間にまとまっています。

```cpp
std::cout   // std（標準）グループの cout
std::endl   // std（標準）グループの endl
std::cin    // std（標準）グループの cin
```

`::` は「〇〇グループの」という意味の区切り記号です。

---

## `using namespace std;` を書くと何が変わる？

毎回 `std::` をつけるのは面倒なので、こう宣言できます。

```cpp
using namespace std;
```

「`std` グループの機能を使うとき、`std::` を省略してもいいよ」という意味です。

```cpp
// using namespace std; なし
std::cout << "Hello!" << std::endl;

// using namespace std; あり
cout << "Hello!" << endl;   // ← すっきり！
```

どちらも全く同じ動きをします。

---

## じゃあ常に書けばいいの？

入門のうちは `using namespace std;` を書いて問題ありません。

ただし、大きなプログラムを書くときは注意が必要です。
`std` 以外のグループにも `cout` という名前があった場合、
`using namespace std;` を書いていると、どちらを使うか曖昧になってしまいます。

そのため、プロのコードでは `using namespace std;` を使わず、
`std::cout` と明示的に書くスタイルが好まれることが多いです。

> 💡 この教材では読みやすさを優先して `using namespace std;` を使っていきます。
> 慣れてきたら `std::` をつけるスタイルも試してみてください。

---

## まとめ

| 書き方 | 意味 |
|--------|------|
| `std::cout` | `std` グループの `cout` を使う |
| `using namespace std;` | `std::` を省略できるようにする |

- `std` はC++標準機能のグループ（名前空間）の名前
- `using namespace std;` を書くと `std::` を省略できる
- どちらを使っても動きは同じ
