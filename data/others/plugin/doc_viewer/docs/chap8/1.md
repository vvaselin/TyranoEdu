## データと処理をひとまとめにしよう

「犬」を表すプログラムを考えてみます。
これまでの方法だと、名前・年齢・鳴き声などを別々の変数で管理することになります。

```cpp
string dogName = "ポチ";
int dogAge = 3;

string catName = "タマ";
int catAge = 5;
```

犬や猫が増えるほど変数が増えて管理しにくくなります。

**クラス** を使うと、関連するデータと処理を1つにまとめられます。

```cpp
Dog dog1;
dog1.name = "ポチ";
dog1.age = 3;
dog1.bark();   // ワン！
```

## クラスの基本の形

```cpp
class クラス名 {
public:
    // メンバ変数（データ）
    型 変数名;

    // メンバ関数（処理）
    戻り値の型 関数名(引数) {
        // 処理
    }
};   // ← セミコロンを忘れずに
```

## はじめてのクラス

```cpp
#include <iostream>
#include <string>
using namespace std;

class Dog {
public:
    string name;
    int age;

    void bark() {
        cout << name << "：ワン！" << endl;
    }

    void introduce() {
        cout << "私は" << name << "、" << age << "歳です。" << endl;
    }
};

int main() {
    Dog dog1;
    dog1.name = "ポチ";
    dog1.age = 3;
    dog1.bark();
    dog1.introduce();

    Dog dog2;
    dog2.name = "ハナ";
    dog2.age = 5;
    dog2.bark();
    dog2.introduce();

    return 0;
}
```

実行結果：
```
ポチ：ワン！
私はポチ、3歳です。
ハナ：ワン！
私はハナ、5歳です。
```

`Dog` という設計図を1つ作るだけで、同じ構造を持つ犬を何匹でも作れます。

## クラスと変数の関係

クラスは**設計図**、そこから作られる実体を **オブジェクト**（またはインスタンス）といいます。

```
class Dog  ← 設計図
   ↓
Dog dog1   ← 設計図から作ったオブジェクト
Dog dog2   ← 同じ設計図から別のオブジェクトも作れる
```

型が `int` や `string` であるように、クラスも1つの「型」として使えます。

---

## コンストラクタ：作るときに初期化する

オブジェクトを作ると同時に値を設定する仕組みが **コンストラクタ** です。
クラス名と同じ名前の特別な関数です。

```cpp
#include <iostream>
#include <string>
using namespace std;

class Dog {
public:
    string name;
    int age;

    Dog(string n, int a) {   // コンストラクタ
        name = n;
        age = a;
    }

    void bark() {
        cout << name << "：ワン！" << endl;
    }
};

int main() {
    Dog dog1("ポチ", 3);   // 作ると同時に name と age を設定
    Dog dog2("ハナ", 5);

    dog1.bark();
    dog2.bark();

    return 0;
}
```

実行結果：
```
ポチ：ワン！
ハナ：ワン！
```

コンストラクタを使うと、初期化し忘れを防げます。

## メンバ関数でデータを操作する

クラスのメンバ関数は、同じクラスのメンバ変数に直接アクセスできます。

```cpp
class Counter {
public:
    int count;

    Counter() {
        count = 0;   // 初期値は0
    }

    void increment() {
        count++;
    }

    void reset() {
        count = 0;
    }

    void show() {
        cout << "カウント: " << count << endl;
    }
};

int main() {
    Counter c;
    c.show();        // → カウント: 0
    c.increment();
    c.increment();
    c.increment();
    c.show();        // → カウント: 3
    c.reset();
    c.show();        // → カウント: 0

    return 0;
}
```

## クラスを使うメリット

| 課題 | クラスで解決できること |
|------|----------------------|
| 関連するデータがバラバラ | メンバ変数としてひとまとめにできる |
| 同じ処理を何度も書く | メンバ関数として1回書けば使い回せる |
| データと処理の対応が不明確 | クラスで「何に対する処理か」が明確になる |

## よくあるミス

### ミス1: クラス定義の末尾の `;` を忘れる

```cpp
class Dog {
    // ...
}   // ← ; がない！エラーになる
```

クラスの `}` の後には必ず `;` が必要です。

### ミス2: `public:` を書き忘れる

```cpp
class Dog {
    string name;   // public: がないと外からアクセスできない
};

Dog dog;
dog.name = "ポチ";   // エラー！
```

`public:` を書かないと、外部からメンバにアクセスできません。

> 💡 `public` / `private` の使い分けはより高度な話題です。最初のうちは `public:` を書く習慣をつけておきましょう。

---

## まとめ

- クラスは**データ（メンバ変数）と処理（メンバ関数）をひとまとめにした設計図**
- クラスから作った実体を**オブジェクト**という
- **コンストラクタ**でオブジェクト作成時に初期値を設定できる
- メンバ変数・関数には `.`（ドット）でアクセスする
- クラス定義の末尾には `;` が必要
