## 目的に合ったコンテナを選ぼう

C++の標準ライブラリには、データをまとめて管理するための型（**コンテナ**）が豊富に揃っています。
`vector` 以外にも、用途に応じて使い分けると便利です。

このページでは代表的なコンテナを紹介します。
「こういうものがある」という概要をつかみ、必要なときに詳しく調べる足がかりにしてください。

---

## pair — 2つの値をセットにする

```cpp
#include <utility>
```

2つの値をペアにして持てます。

```cpp
#include <iostream>
#include <utility>
using namespace std;

int main() {
    pair<string, int> p = {"Alice", 90};

    cout << p.first << endl;    // → Alice
    cout << p.second << endl;   // → 90

    return 0;
}
```

`make_pair()` でも作れます。

```cpp
auto p = make_pair("Bob", 75);
```

`vector` と組み合わせてよく使います。

```cpp
vector<pair<string, int>> scores;
scores.push_back({"Alice", 90});
scores.push_back({"Bob", 75});

for (auto& s : scores) {
    cout << s.first << ": " << s.second << endl;
}
```

---

## tuple — 3つ以上の値をセットにする

```cpp
#include <tuple>
```

`pair` の拡張版で、3つ以上の値をまとめられます。

```cpp
#include <iostream>
#include <tuple>
using namespace std;

int main() {
    tuple<string, int, double> t = {"Alice", 20, 170.5};

    cout << get<0>(t) << endl;   // → Alice
    cout << get<1>(t) << endl;   // → 20
    cout << get<2>(t) << endl;   // → 170.5

    return 0;
}
```

要素へのアクセスは `get<インデックス>(変数)` を使います。

> 💡 C++17以降では **構造化束縛** で分かりやすく取り出せます。
> ```cpp
> auto [name, age, height] = t;
> cout << name << endl;   // → Alice
> ```

## map — キーと値の対応（辞書）

```cpp
#include <map>
```

「キー → 値」の対応を管理します。辞書のようなイメージです。
キーは自動的に**昇順でソート**されます。

```cpp
#include <iostream>
#include <map>
using namespace std;

int main() {
    map<string, int> scores;

    scores["Alice"] = 90;
    scores["Bob"]   = 75;
    scores["Carol"] = 85;

    cout << scores["Alice"] << endl;   // → 90
    cout << scores["Bob"] << endl;     // → 75

    // 全要素を出力（キーの昇順）
    for (auto& [key, val] : scores) {
        cout << key << ": " << val << endl;
    }

    return 0;
}
```

実行結果：
```
90
75
Alice: 90
Bob: 75
Carol: 85
```

### キーの存在確認

```cpp
if (scores.count("Alice")) {
    cout << "Aliceのデータがあります" << endl;
}

if (scores.find("Dave") == scores.end()) {
    cout << "Daveのデータはありません" << endl;
}
```

## unordered_map — 順序なしの高速map

```cpp
#include <unordered_map>
```

`map` と使い方はほぼ同じですが、内部の仕組みが違います。

| | `map` | `unordered_map` |
|-|-------|-----------------|
| 順序 | キーで昇順ソート | 順序なし |
| 検索速度 | O(log n) | O(1)（平均） |
| 用途 | 順序が必要なとき | 高速なとき |

```cpp
unordered_map<string, int> scores;
scores["Alice"] = 90;
cout << scores["Alice"] << endl;   // → 90
```

## set — 重複なしのコレクション

```cpp
#include <set>
```

同じ値を1つしか持てないコレクションです。要素は**自動でソート**されます。

```cpp
#include <iostream>
#include <set>
using namespace std;

int main() {
    set<int> s;
    s.insert(5);
    s.insert(3);
    s.insert(5);   // 重複は無視される
    s.insert(1);

    for (int x : s) {
        cout << x << endl;
    }

    return 0;
}
```

実行結果：
```
1
3
5
```

### 要素の存在確認・削除

```cpp
if (s.count(3)) {
    cout << "3はあります" << endl;
}

s.erase(3);   // 3を削除
```

## queue — キュー（先入れ先出し）

```cpp
#include <queue>
```

先に入れた要素が先に出てくる（FIFO: First In, First Out）データ構造です。
「順番待ち列」のイメージです。

```cpp
#include <iostream>
#include <queue>
using namespace std;

int main() {
    queue<int> q;
    q.push(1);
    q.push(2);
    q.push(3);

    while (!q.empty()) {
        cout << q.front() << endl;   // 先頭を見る
        q.pop();                     // 先頭を取り出す
    }

    return 0;
}
```

実行結果：
```
1
2
3
```

---

## stack — スタック（後入れ先出し）

```cpp
#include <stack>
```

後に入れた要素が先に出てくる（LIFO: Last In, First Out）データ構造です。
「積み上げた本の山」のイメージです。

```cpp
#include <iostream>
#include <stack>
using namespace std;

int main() {
    stack<int> st;
    st.push(1);
    st.push(2);
    st.push(3);

    while (!st.empty()) {
        cout << st.top() << endl;   // 一番上を見る
        st.pop();                   // 一番上を取り出す
    }

    return 0;
}
```

実行結果：
```
3
2
1
```

## コンテナ選びの指針

| やりたいこと | 使うコンテナ |
|------------|-------------|
| 可変長の配列 | `vector` |
| 2つの値をセット | `pair` |
| 3つ以上の値をセット | `tuple` |
| キーで値を引く（順序あり） | `map` |
| キーで値を引く（高速） | `unordered_map` |
| 重複なしで管理 | `set` |
| 先入れ先出し | `queue` |
| 後入れ先出し | `stack` |

---

## まとめ

- `pair` — 2値セット（`.first` / `.second`）
- `tuple` — 多値セット（`get<N>()`）
- `map` — キー→値の辞書（ソート済み）
- `unordered_map` — 高速な辞書（順序なし）
- `set` — 重複なし・自動ソート
- `queue` — 先入れ先出し
- `stack` — 後入れ先出し
